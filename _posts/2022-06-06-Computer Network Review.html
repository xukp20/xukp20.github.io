3.5 数字调制与多路复用

传输方式：

  • 基带传输：未对载波调制
  • 频带传输：使用调制解调器，搬离了原本的信号频谱

编码技术：

  • 不归零编码：低电平0，高电平1
    • 难以分辨一位的边界，需要双方时钟同步
  • 曼彻斯特码：相位编码
    • 每一个位中间跳变，低到高为0，高到低为1
    • 中间跳变既为数据又为时钟，自同步
  • 差分曼彻斯特编码：
    • 每一位中间会发生跳变，每位开始有跳变为0，无为1
    • 时钟数据分离，便于提取

频带传输

用基带信号对载波进行调制，变为适合传送的信号

调制技术：调幅，调频，调相

复用技术

多个用户使用一个共享信道，避免干扰

  • 频分复用FDM：将多路基带调制到不同频率载波上
  • 波分复用WDM：利用多个激光器在同一光纤发送激光
  • 时分复用TDM：划分时间为时分复用帧，利用率不高，经常出现空隙
  • 统计时分复用STDM：动态按需分配共用信道

码分复用 Code Division Multiple Access

码分多址CDMA依靠不同的地址码区分地址，

各个站的码片序列彼此正交

收到的码片序列是所有同时发送的码片的和，分别与每个站的码片序列内积即可知是否发送及内容

Ch04 数据链路层

NIC：Network Interface Card 一般实现了物理层和一部分数据链路层功能

关键假设

  • 分层进程独立
  • 提供可靠服务：要提供可靠的面向连接的服务，能从网络层获得
  • 只处理通信错误

数据链路层提供的服务

  • 无确认 无连接
    • 适用误码率低的可靠信道，实时通信
    • 以太网
  • 有确认 无连接
    • 适用不可靠的信道，无线信道
    • 802.11
  • 有确认 有连接
    • 适用：长延迟的不可靠信道

数据链路层功能

成帧

将物理层的比特流划分成帧，从而实现错误的检测和纠正，借助帧的各个域实现：帧同步 帧定界 问题

成帧方式：

  • 字节计数法
  • 带字节填充的定界符法
  • 带比特填充的定界符法
  • 物理层编码违例

差错检测和纠正

解决方案：

  • 确认：防止差错
  • 定时器：防止丢失
  • 顺序号：防止乱序，重复

检错码

发现错误后重传，用于高可靠、误码率低的信道上

  • 奇偶校验
    • 一位校验位，校验1的数量
    • 偶校验：保证1的个数为偶数个，加上校验位
    • 没法处理连续的差错
  • 校验和
    • TCP/IP: 16位二进制补码求和，结果取反为检验和；接收方进行求和应为全1
  • 循环冗余校验CRC
    • 原始数据D：k位；n+1位二进制多项式G产生n位CRC；
    • D左移k位后除以G，余数R为效验码
    • 能检测少于 n + 1位的突发错误

纠错码

又称为 前向纠错 FEC，意思是不需要反馈错误信息，接受方就可以纠错

使用重传叫做后向纠错

  • 海明码
    • 校验位是2的幂次方位，数据依次放入其他位
    • 四位海明码： 1 - 15，11个数据位，1,2,4,8校验
    • 数据位k影响其因子的校验位值
    • 缺省为偶校验
    • 填充方法

4.3 基本的数据链路层协议

乌托邦式单工协议P1

  • 单工
  • 完美信道：无丢失和错误
  • 始终就绪：网络层随时提供数据
  • 立即完成：物理层能够瞬间处理好数据

无流量控制和纠错，近似于无确认的无连接服务

问题：

  • 发送过快导致接收方被淹没

无错信道上的停等式协议P2

  • 数据传输依然是单向，但物理信道是半双工的
  • 停等协议：增加确认机制，哑帧确认
  • 引入了ACK

问题：

  • 可能发生丢包
  • 可能损坏

有错信道上的单工停等式协议P3

  • 为了解决丢失问题：引入计时器
  • 为了解决重复问题：引入帧序号，ACK携带下一个希望得到的帧序号

效率评估： 发送时间：F/R 空闲时间 RTT = 2I

利用率为

F = RD, 50%

问题：

  • 一次只能有一个帧在发送中，空闲时间长

解决方案：

  • 增长帧大小：导致更多错误，重传增多，不可行
  • 发送多个帧

4.4 滑动窗口协议

发送方能够一次发送多个，暂存以便重传，

一比特滑动窗口协议 P4

  • 双工协议：需要全双工信道
  • 引入捎带确认
  • 仍为停等，只是双向
  • 发送方和接收方的逻辑相同，接受窗口和发送窗口都为1
  • 问题：同时开始发送会有一半重复

回退N协议 P5

  • 简化设计：接收窗口为1
  • 出错后面所有帧全部重发，超时重传
  • 窗口限制：
    • 接收窗口为1
    • 发送窗口为 Wt ≤ 2^n - 1，考虑ACK全部丢失情况
  • 引入 累计确认
  • 反向确认：NACK(k) 期望接受k

选择重传协议 P6

  • 接收方有大于1的接收窗口，暂存出错帧后面的帧
  • 每个PDU需要一个计时器
  • NAK：遇到坏帧时返回否认确认帧
  • 窗口限制 Wt ≤ 2^(n-1)

CSMA 载波侦听多路访问协议

持续式CSMA

  • 侦听时若空闲则发送
  • 发送时发生冲突：等待随机时间后再次侦听
  • 介质忙：持续监听至空闲，发送
    • 只要有两个要发送，就会冲突

非持续式CSMA

  • 指的是不持续侦听，如果发现介质忙则随机时间后再侦听
    • 利用率下降
    • 同时侦听到空闲时依然会冲突

p-持续式 CSMA

  • 侦听到空闲时以p概率发送，（1-p）延迟一个时间单元再次侦听
  • 介质忙则持续侦听
  • p指的是空闲时发送的概率，依然持续监听

生成树协议

结构选择：

  • 树状结构：不可靠
  • 冗余拓扑：可能存在环路
    • 环路导致MAC地址表不稳定
    • 环路导致重复帧
    • 环路导致广播风暴

生成树选举过程

  • 选举根桥：BPDU桥协议数据单元中优先级最小，MAC地址最小的成为根桥
    • 根桥的所有端口都处于转发状态，所有端口都是指定端口
  • 每个非根桥选出一个根端口：比较自己所有端口到根桥的路径开销，选最小的
    • 根端口处于转发状态
    • 开销为路径上所有链路开销之和，与速率有关
  • 每个网段确定一个指定端口：从所有连接它的网桥的端口中选择最小根路径开销的一个，处于转发状态；其他端口全部阻塞

状态迁移：

源路由网桥

  • 广播发现帧寻找最佳路由

链路层交换机

交换模式

  • 存储转发：接收整个帧并进行CRC校验之后转发
    • 延迟大
    • 不转发错帧，支持非对称交换
  • 直通交换：接收到目的MAC之后就开始转发
    • 延迟小
    • 转发错误帧，不支持非对称交换
  • 无碎片交换：接收到前64字节后开始转发
    • 过滤冲突碎片，延迟和错误转发介于以上二者之间

DHCP 动态主机配置协议

工作模式：C/S

  • 服务器的报文的MAC地址是单播地址；用户机的MAC是广播地址；两者的目标IP地址都是广播

ARP地址解析协议

如果两个主机在同一个IP网络，则只要知道B的MAC，就直接当作目的MAC封装到链路层帧通过链路层发送

IP包转发

  • 直接交付：源与目的在同一IP子网，不用路由器，只用交换机在链路层
    • 目的MAC直接是B的MAC
    • 目的IP是B的IP
  • 间接交付：不在同一IP子网，使用路由器转发
    • 目的MAC是路由器对应端口的MAC
    • 目的IP是B的IP
    • 路由器转发时会修改源MAC和目的MAC，不修改IP
